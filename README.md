# ISTQB CTFL 랜덤 문제 풀이

KSTQB 공개 샘플 문제(A·B·C·D)에서 추출한 **186문제**로 만든 랜덤 풀이 사이트입니다.
실러버스 v4.0.1 기반이며, 챕터별 필터·오답노트·즉시 해설을 지원합니다.

## 파일 구성

```
index.html       풀이 화면 (모든 UI/로직)
questions.json   문제 데이터 (186문제)
```

`index.html`이 같은 폴더의 `questions.json`을 `fetch`로 읽어옵니다.

## 띄우는 방법

### GitHub Pages (권장)
1. 이 폴더의 두 파일을 GitHub 저장소에 올립니다.
2. 저장소 **Settings → Pages → Branch: main / root** 로 설정합니다.
3. 몇 분 뒤 `https://<아이디>.github.io/<저장소>/` 로 접속하면 됩니다.

### 로컬에서 바로 확인
`index.html`을 더블클릭으로 여는 방식은 브라우저 보안정책(CORS) 때문에
`questions.json`을 못 읽습니다. 아래처럼 간단한 서버로 띄우세요.

```bash
cd 이폴더
python -m http.server 8000
# 브라우저에서 http://localhost:8000 접속
```

## 나중에 AI로 만든 문제 추가하기

`questions.json`은 아래 형식의 객체 배열입니다. 같은 형식으로 항목을 추가하면
사이트가 자동으로 풀이에 섞어줍니다. (코드 수정 불필요)

```json
{
  "id": "AI-001",            // 고유 ID (겹치지 않게)
  "set": "AI",               // 출처 라벨 (태그에 표시됨)
  "num": "1",                // 문제 번호
  "extra": false,            // 추가문제 여부 (표시용)
  "stem": "문제 지문...",
  "options": {               // 보기 (a~e 중 필요한 만큼)
    "a": "보기 a",
    "b": "보기 b",
    "c": "보기 c",
    "d": "보기 d"
  },
  "answer": ["c"],           // 정답 (복수정답이면 ["a","e"] 처럼)
  "lo": "FL-1.1.1",          // 학습목표 (태그 표시용, 임의값 가능)
  "chapter": "1",            // 챕터 번호 1~6 (챕터 필터에 사용)
  "klevel": "K2",            // K-레벨 (태그 표시용)
  "explanation": "해설..."   // 정답/오답 해설
}
```

핵심 필드는 `stem`, `options`, `answer`, `chapter` 입니다.
`chapter`만 1~6 중 하나로 정확히 넣으면 챕터 필터가 작동합니다.

여러 파일로 나누고 싶다면 AI 문제를 별도 `ai-questions.json`으로 만든 뒤
`index.html`에서 두 파일을 합쳐 불러오도록 `load()` 함수만 살짝 고치면 됩니다.

## 기능

- **챕터 필터** — 원하는 챕터만 골라 랜덤 출제 (안 고르면 전체)
- **문제 수 / 순서** — 10·20·40·전체, 랜덤 또는 순서대로
- **즉시 해설** — 보기마다 왜 맞고 틀린지 상세 해설 표시
- **오답노트** — 틀린 문제는 자동 저장(localStorage), 다시 풀어서 맞히면 제거
- **결과 요약** — 점수·정답률·합격선(65%) 표시, 방금 틀린 문제 바로 복습
